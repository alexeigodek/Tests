Exercise 1 — Overview

Demo Setup & Platform Tour

Get into the Global SC Demo Instance via Okta SSO and walk through the six platform sections you'll navigate during every demo.

Access Method
Okta SSO
Duration
~10 minutes
Platform Sections
6 sections

What You'll Cover

This exercise walks through getting into the demo environment and orienting a prospect on the six key sections of the Boomi MFT platform. These sections form the backbone of every flow walkthrough — understanding them here makes the rest of the demo feel natural.

The platform comes pre-loaded with continuous synthetic activity across all six flows, so it looks live and healthy from the moment you log in — no setup required.

No separate credentials needed. Access is single-click from Okta. The Global SC Demo tile should already be assigned to your apps.
Step 1 of 4

Log In & Land on Dashboard

One-click SSO from Okta — no separate credentials required.

There are multiple MFT tiles in Okta. Use the tile named exactly "Boomi MFT - SE Demo" — not the other MFT tiles. Using the wrong tile will land you in a different environment.
  1. 1

    Open your Okta portal. You will see multiple Boomi MFT tiles — look for the one named "Boomi MFT - SE Demo".

    📸 Screenshot: Okta portal — select the tile named "Boomi MFT - SE Demo"

  2. 2

    Click the "Boomi MFT - SE Demo" tile. SSO takes you directly into the Global SC Demo Instance — you'll land authenticated with full access. No username or password prompt.

  3. 3

    Confirm you're on the Dashboard. Continuous activity should be visible immediately — synthetic files are cycling through all six flows in the background, making the platform look live without any manual setup.

  4. 4

    Navigate to the Overview page to see Ingress, Egress, and storage metrics. Hover over Egress to split cloud vs. runtime values — this is the MFT pricing metric and a good early talking point for pricing conversations.

Demo tip: The continuous activity on the Dashboard is one of the strongest first impressions in the demo. Let the prospect notice the live movement before explaining it — then reveal the synthetic loop as a feature, not a workaround.
Step 2 of 4

Dashboard & Overview Metrics

The first two sections prospects see — set the tone for a healthy, active platform.

Dashboard

The Dashboard is your starting point for every demo. It shows continuous activity — synthetic files moving through all flows. This demonstrates the platform is live and healthy before you've done anything.

Key talking point: the Dashboard gives operations teams real-time visibility across all transfer activity. No need to drill into individual flows to know the platform is healthy.

Overview

The Overview page surfaces three key metrics — Ingress, Egress, and Storage — aggregated across all flows.

Pricing talking point: Egress is the MFT pricing driver. When you hover over Egress in the Overview, it splits into Cloud Egress vs. Runtime Egress. Use this to explain the pricing model — customers pay for data moving out through cloud-hosted endpoints, not through on-prem runtime agents.
Step 3 of 4

Organizations & Flows

The structural backbone — who participates, and how files move between them.

Organizations

Organizations represent the entities taking part in file transfers — partners, banks, HQ, healthcare providers, automotive suppliers. Each org can have its own Endpoints, Users, Keys, and Nodes.

The _example organization has pre-configured sample endpoints — useful for showing Test Connection during the demo without touching live partner data.

Use the _example org to validate connections live during the demo — click any endpoint and hit Test Connection. This is a high-impact moment: real-time connectivity check in the UI.

Flows

Flows are the core objects — each one defines a transfer pattern with a source, one or more targets, and optional processing steps like encryption, AV scanning, renaming, and path mapping.

Each flow has a configurable upload widget that can be exported to FileZilla for an SFTP-native demo path. This is especially useful for prospects who want to see the full SFTP experience rather than browser-based upload.

Step 4 of 4

Activity & Alerts

The audit trail and monitoring story — two of the strongest closing sections.

Activity

Activity is the full audit trail for every transfer. Filter by flow, expand rows to reveal both source and target sides, and click any state badge to view the complete file lifecycle.

This is where the demo becomes tactile — upload a file in Flow 01 or 05, then immediately switch to Activity to show the customer their file moving in real time through Upload → AV Scan → Encrypt → Route → Ready for Pickup.

Alerts

Alerts has been pre-populated with two demo fixtures — an Inactivity Alert and a Connection Error Alert. These are not live alerts — they're demo fixtures configured to tell a monitoring story.

Do NOT clear the pre-loaded alerts. They are demo fixtures. Clearing them removes a key monitoring talking point from the demo and they cannot be easily restored.

Inactivity Alert: Fires when a flow hasn't processed a file within an expected time window. Great for SLA monitoring conversations — "what if a partner stops sending?"

Connection Error Alert: Fires when a configured endpoint becomes unreachable. Shows proactive partner connectivity monitoring — no need to wait for a partner to report a problem.

Alert configuration lives inside the flow builder, per endpoint. Walk through this when discussing security or monitoring during the Flow 01 source walkthrough.
