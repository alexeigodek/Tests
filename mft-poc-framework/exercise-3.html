Step 1 of 2

Troubleshooting & Error Handling

Diagnose MFT failures quickly using error codes, connectivity tools, and the audit trail.

Error Code Reference

Code Class Meaning Common Causes First Steps
4xx Client Error The client request has a problem Bad credentials, missing file, wrong remote path, insufficient permissions, account locked Verify username/password or SSH key; check remote path exists; confirm account is active
5xx Server Error The server cannot fulfill the request Server unavailable, disk full on remote host, internal MFT configuration issue, service restart in progress Check server availability (ping / telnet); verify MFT service status; check disk space on server

Connectivity Verification Tools

Before debugging credentials or configurations, verify the network path is open. Use these commands from the Atom host machine or a machine in the same network segment:

# Verify the port is reachable (SFTP = 22, FTPS explicit = 21, FTPS implicit = 990)
telnet <remote-host> <port>

# Basic reachability check
ping <remote-host>

# From Linux: test SFTP connection with verbose output
sftp -v -i ~/.ssh/id_rsa user@remote-host

# Check if a specific port is open (Linux)
nc -zv <remote-host> 22
Always run connectivity checks from the Boomi Atom's host machine, not your laptop. Firewalls are applied between the Atom and the remote host — your laptop network may behave differently.

The MFT Audit Trail

Every file transfer in Boomi MFT moves through four states. When a transfer fails, identifying which state it failed at tells you immediately where to look:

Received
File arrived at the MFT endpoint and logged
Validated
Auth checked, permissions verified
Processed
Workflow / routing rules applied
Delivered
File confirmed at final destination
Failure at Stage What It Means Where to Look
Received File arrived but was rejected at ingestion Check file size limits, MFT listener configuration, port availability
Validated Auth or permission check failed Check credentials, SSH key fingerprint, Virtual Folder permissions, account status
Processed Routing or transformation rule failed Check workflow configuration, filename filters, any transformation scripts
Delivered Destination unreachable or write failed Check target server connectivity, disk space, destination path permissions
Demo move: Intentionally fail a transfer during the POC (wrong password, blocked path) and show the customer exactly where it appears in the Activity Log. This is more convincing than a flawless demo — it shows you understand the failure modes.
