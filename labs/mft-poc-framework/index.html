MFT POC Framework

Master the core concepts, security patterns, and three fundamental POC scenarios that cover 90% of real-world Boomi MFT customer use cases. Built for SEs preparing to run technical POCs.

Modules

1

Protocol & Security Fundamentals

Understand FTP vs. FTPS vs. SFTP at a technical level, compare authentication methods, and learn how Boomi's MFT architecture handles encryption in transit, at rest, and through a DMZ proxy.

SFTP / FTPS / FTP SSH Keys X.509 Certs DMZ
~25 min 3 steps
2

The 3 Core POC Patterns

Walk through the three patterns that cover 90% of customer scenarios: Internal Orchestration (agent-based routing), MFT as Client (scheduled pull/push), and MFT as Server (SFTP listener with virtual folders).

Internal Orchestration MFT as Client MFT as Server Virtual Folders
~40 min 3 steps
3

Troubleshooting & Mock POC Validation

Learn the MFT audit trail (Received → Validated → Processed → Delivered), decode 4xx and 5xx error codes, and complete the interactive Mock POC Technical Checklist to validate your SE readiness.

Error Handling Activity Log POC Checklist
~25 min 2 steps

Prerequisites

  • Access to a Boomi trial or sandbox environment with MFT enabled
  • Basic familiarity with Boomi Integration (process building) is helpful but not required
  • An SFTP client installed locally (FileZilla or WinSCP recommended)
  • A VM (Linux or Windows) available to deploy a Boomi Atom for Pattern 1
  • AWS S3 bucket or partner SFTP server credentials for Pattern 2 (optional)
